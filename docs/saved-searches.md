# Сохранённые заявки на командировку

## Что это такое

Заявка на командировку — это сохранённые параметры поиска маршрута для группы сотрудников. Пользователь сначала выполняет обычный поиск, затем вручную сохраняет параметры как заявку и может позже повторно проверить маршруты и наличие мест.

Автоматическая периодическая проверка на этом этапе не запускается: MVP поддерживает только ручное действие «Проверить сейчас».

## Структура данных

Сущность `SavedSearch` хранит только параметры маршрута и агрегированную статистику последней проверки:

- `id` — UUID заявки;
- `title` — пользовательское название;
- `origin`, `destination`, `departure_date` — направление и дата;
- `passengers` — количество сотрудников;
- `allowed_transport` — разрешённые виды транспорта;
- `max_transfers`, `minimum_transfer_minutes` — ограничения пересадок;
- `preferred_classes`, `require_group_together`, `allow_split_group` — настройки доступности;
- `monitoring_enabled` — флаг будущего мониторинга;
- `created_at`, `updated_at` — UTC-временные метки;
- `last_checked_at` — время последней ручной проверки;
- `last_check_status` — `never_checked`, `checking`, `routes_found`, `no_available_routes` или `failed`;
- `last_routes_count`, `last_available_routes_count` — краткая статистика;
- `last_error` — текст ошибки последней неуспешной проверки.

Паспортные данные, билеты, платежные данные и реальные пользовательские документы не сохраняются.

## API endpoints

Все endpoints находятся под `/api/v1/saved-searches`:

- `POST /api/v1/saved-searches` — создать заявку из параметров существующего поиска и необязательного `title`;
- `GET /api/v1/saved-searches` — получить список заявок;
- `GET /api/v1/saved-searches/{id}` — открыть одну заявку;
- `PATCH /api/v1/saved-searches/{id}` — изменить название, `monitoring_enabled` и безопасные параметры маршрута;
- `DELETE /api/v1/saved-searches/{id}` — удалить заявку;
- `POST /api/v1/saved-searches/{id}/check` — вручную повторно проверить маршруты.

Существующий endpoint `POST /api/v1/routes/search` не изменяется и продолжает возвращать обычные результаты поиска.

## Как выполняется ручная проверка

`SavedSearchService` получает сохранённую заявку, переводит её в существующую модель `RouteSearchRequest` и запускает текущий `RouteSearchService`/`RouteEngine`. Алгоритм поиска маршрутов не дублируется.

Во время проверки статус временно становится `checking`. После успешной проверки сервис обновляет:

- `last_checked_at`;
- `last_routes_count`;
- `last_available_routes_count`;
- `last_check_status`.

Если возникает исключение, сервис сохраняет текст в `last_error`, ставит статус `failed` и не оставляет заявку в состоянии `checking` навсегда.

## Что означает monitoring_enabled сейчас

`monitoring_enabled` — это пользовательская настройка будущего автоматического мониторинга. Сейчас она только сохраняется, отображается в интерфейсе и может быть включена или приостановлена пользователем. Фоновый worker, Redis, cron-задачи и Telegram-уведомления намеренно не добавляются.

## Временное файловое хранилище

`FileSavedSearchRepository` хранит заявки в JSON-файле. Путь задаётся переменной окружения `SAVED_SEARCHES_FILE`; значение по умолчанию для локальной разработки — `data/saved-searches.json`.

Репозиторий автоматически создаёт директорию и пустой JSON-файл, корректно читает отсутствующий или пустой файл и записывает изменения безопасно: сначала во временный файл, затем заменяет основной файл.

## Ограничения Render

На бесплатных/эфемерных окружениях Render локальная файловая система, включая `/tmp` и файлы внутри сервиса, не является постоянным хранилищем. Сохранённые заявки могут исчезнуть после redeploy, рестарта сервиса или пересоздания инстанса. Это временное MVP-хранилище, не предназначенное для реальных пользовательских данных.

## Как заменить на PostgreSQL позже

Следующий шаг — добавить реализацию `SavedSearchRepository` поверх PostgreSQL и подключить её через конфигурацию приложения. API, сервисный слой и frontend можно оставить прежними: потребуется заменить только реализацию методов `create`, `list`, `get`, `update`, `delete` и добавить миграции схемы.
