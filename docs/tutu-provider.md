# Адаптер Туту

## Назначение

Адаптер `tutu` подготавливает антикоррупционную границу для будущего официального B2B API Туту. Он нужен, чтобы приложение могло работать с поездами, вагонами, классами обслуживания, доступными местами, номером купе, номером вагона, верхними и нижними местами, боковыми местами и ограничениями мужского или женского купе через единые модели проекта.

## Почему реальные запросы пока не выполняются

Официальная документация B2B API Туту в репозитории отсутствует. Поэтому в коде нет реального HTTP-клиента, endpoints, HTML parsing, scraping, Playwright или Selenium. До получения документации используется только `MockTutuClient` с детерминированными сценариями.

## Какие данные ожидаются от официального B2B API

Нужны read-only данные о расписании поездов, списке вагонов, классе обслуживания, количестве доступных мест, схеме мест, типе места, верхнем или нижнем положении, боковых местах, номере купе, номере вагона и ограничениях пола.

## Какие credentials могут потребоваться

Конфигурация поддерживает безопасные пустые переменные: `TUTU_BASE_URL`, `TUTU_API_KEY`, `TUTU_LOGIN`, `TUTU_PASSWORD`, `TUTU_TIMEOUT_SECONDS` и `TUTU_ENABLED`. Секреты не должны попадать в Git, API, логи, ошибки или `render.yaml`.

## Как заменить MockTutuClient на официальный клиент

После получения официальной документации нужно реализовать клиент, который соответствует `TutuClient`, и передать его в `TutuAvailabilityProvider`. Методы клиента должны описывать бизнес-возможности, а не выдуманные URL.

## Какие методы нужно сопоставить

Нужно сопоставить `search_trains`, `get_train_carriages` и `get_carriage_places` с официальными read-only методами Туту. Названия DTO в проекте внутренние и не утверждают формат официального API.

## Как работают SeatPreferences

`TutuAvailabilityProvider` преобразует DTO в `RailwayPlace` и передаёт места в `SeatAllocationService`. Сервис применяет `SeatPreferences`: нижние или верхние места, одно купе, пустое купе без посторонних, один вагон, соседние места, исключение боковых мест и ограничения пола.

## Что приложение не делает

Приложение не покупает, не бронирует и не оформляет билеты. Адаптер предназначен только для подготовки чтения расписания и доступности мест.

## Почему нельзя придумывать endpoint

Выдуманные endpoints создают ложную интеграцию, могут нарушить правила сервиса и усложняют последующее подключение официального API. До документации допустимы только protocol, DTO, mapper и mock-клиент.
